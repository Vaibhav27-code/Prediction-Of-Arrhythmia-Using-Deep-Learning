# 🫀 Prediction of Arrhythmia Using Deep Learning

A web-based ECG arrhythmia detection system built with Flask and TensorFlow/Keras. Users enter key ECG interval measurements and the system predicts whether the pattern indicates **Normal** sinus rhythm or **Arrhythmia**, along with personalized health suggestions.

## About the Project

Arrhythmia is an irregular heartbeat condition that affects millions worldwide. Early detection is critical to preventing serious cardiac events. This project combines a **deep learning neural network** trained on real ECG data (MIT-BIH Arrhythmia Database) with a **clinical rules engine** to provide reliable, interpretable predictions through a simple web interface.

The model takes 5 ECG interval measurements as input and outputs a binary prediction (Normal / Arrhythmia) along with health recommendations.

## How It Works

```
User enters ECG values
        │
        ▼
   Validate input ranges
        │
        ▼
  Feature Remapping         ← Converts raw durations to match training distribution
        │
        ├──► Clinical Rules Engine   ─► Clear Arrhythmia / Normal → return result
        │         (RR, QRS, QTc)
        │
        └──► Neural Network Model   ─► Probability score (sigmoid)
                                              │
                                         Combine votes
                                              │
                                         Final Prediction
                                              │
                                     Personalised Suggestion
```

## ECG Features Used

| Feature | Description | Normal Range |
|---|---|---|
| **RR Interval** | Time between successive R-peaks | 0.6 – 1.2 s |
| **P Wave** | Atrial depolarisation duration | 0.05 – 0.25 s |
| **QRS Complex** | Ventricular depolarisation | 0.05 – 0.12 s |
| **T Wave** | Ventricular repolarisation | 0.05 – 0.40 s |
| **QT Interval** | Total ventricular activity | 0.20 – 0.70 s |

---

## Clinical Rules Engine

In addition to the neural network, a rule-based engine applies standard cardiology thresholds:

| Condition | Threshold | Interpretation |
|---|---|---|
| Tachycardia | RR < 0.60 s | HR > 100 bpm → Arrhythmia |
| Bradycardia | RR > 1.20 s | HR < 50 bpm → Arrhythmia |
| Wide QRS | QRS > 0.12 s | Bundle branch block / VT → Arrhythmia |
| Prolonged QTc | QTc > 0.45 s | Long QT syndrome → Arrhythmia |

Clinical rules take full priority over the model for clear-cut cases. The model is the decider only when the case is ambiguous.

## Setup & Installation

### Prerequisites

- Python 3.10 or 3.11
- pip

### 1. Clone the repository

```bash
git clone https://github.com/Vaibhav27-code/Prediction-Of-Arrhythmia-Using-Deep-Learning.git
cd Prediction-Of-Arrhythmia-Using-Deep-Learning
```

### 2. Create and activate a virtual environment

```bash
python -m venv venv

# Windows
venv\Scripts\activate

# macOS / Linux
source venv/bin/activate
```

### 3. Install dependencies

```bash
pip install flask flask-login flask-sqlalchemy flask-wtf tensorflow scikit-learn numpy joblib python-dotenv wfdb scipy
```

### 4. Create `.env` file

Create a `.env` file in the project root:

```env
SECRET_KEY=your-secret-key-here
FLASK_DEBUG=true
```

## Training the Model

> ⚠️ The trained model files (`cnn_ecg_model.h5`, `scaler.pkl`) are not included in the repo due to file size. You must train the model before running the app.

```bash
python train_model.py
```

This will:
1. Automatically download all 48 MIT-BIH Arrhythmia Database records from PhysioNet
2. Extract 5 ECG features per beat using geometric signal analysis
3. Balance the dataset (1:1 Normal:Arrhythmia)
4. Train a Dense neural network with BatchNorm + Dropout
5. Save `cnn_ecg_model.h5` and `scaler.pkl`

Training typically takes **5–15 minutes** depending on your internet speed and hardware.
